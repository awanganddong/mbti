# MBTI / SBTI 微信小程序

原生微信小程序实现（无 React / Next.js）。使用 **微信开发者工具** 打开本项目根目录即可预览与上传。

## 目录说明

- `app.js` / `app.json` / `app.wxss`：小程序入口与全局样式  
- `pages/`：页面（首页、MBTI 测试与结果、历史、SBTI）  
- `data/`：题目与类型数据（CommonJS）  
- `utils/`：计分、本地存储、SBTI 算法  
- `images/mbti/`：16 型配图（本地资源）  
- `project.config.json`：项目配置；请将 `appid` 改为你自己的小程序 AppID  

## 功能

- **MBTI**：答题 → 本地存储结果 → 完整类型解读页  
- **MBTI 历史**：查看过往记录并进入对应结果  
- **SBTI**：十五维度趣味测试（配图使用上游仓库 CDN）  

## 数据说明

MBTI 题库与类型文案来自原项目数据；SBTI 逻辑与 [SBTI-test](https://github.com/UnluckyNinja/SBTI-test) 一致，仅供娱乐。

## 无效代码已移除

已删除原 Web 技术栈相关文件（Next.js、Chakra、React 页面与组件、TypeScript 源码数据等），仅保留小程序运行所需 JS 与资源。

## 模拟器报错：fork process timeout

该提示来自**微信开发者工具**在拉起模拟器子进程时超时，多数与**本机环境**有关，而不是业务代码语法错误。可按顺序尝试：

1. **完全退出**微信开发者工具后重新打开，再点「编译」。
2. 菜单 **工具 → 清除缓存 → 清除全部缓存**，然后重启工具。
3. 将微信开发者工具**更新到最新稳定版**（旧版在 macOS 上偶发 fork 超时）。
4. 系统资源：关闭占用内存/CPU 过多的其它软件；若使用公司安全软件，可暂时将开发者工具加入白名单。
5. **更换基础库版本**：右上角「详情 → 本地设置」中切换模拟器基础库为其它版本试一次。
6. 若仍失败：在 **详情 → 本地设置** 中关闭「启用多核心编译」「上传代码时样式自动补全」等可选增强项后再试。

本项目已在 `project.config.json` 中关闭增强编译与开发态压缩，以减轻首次编译负担；发布前可在 `setting` 里再把 `minified` / `minifyWXSS` / `minifyWXML` 改为 `true` 做体积优化。
